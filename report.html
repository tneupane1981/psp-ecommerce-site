1. Conceptual Site Overview

The Conceptual Site Overview is documented at concept.html and is accessible from the main navigation bar (Concept link) and from the footer Quick Links on every page of the site.

Purpose

Pet Supply Plus is a full-featured e-commerce website designed to provide pet owners with a convenient, trustworthy, and enjoyable online shopping experience. The site offers a wide selection of premium pet supplies covering dogs, cats, birds, fish, reptiles, and small animals — all in one place. The primary purpose is to replicate the feel of a professional retail pet supply store online, complete with product browsing, a shopping cart, checkout flow, customer service, and informational pages.

Target Audience

  • Pet Owners: Individuals who own one or more pets and regularly purchase food, accessories, and care products.
  • New Pet Parents: First-time pet owners looking for guidance, starter kits, and essential supplies.
  • Gift Shoppers: People shopping for pet-related gifts for friends and family.
  • Veterinary Clients: Customers referred by vets seeking specific health or dietary products.

Goals

  • Provide a seamless, end-to-end shopping experience from product discovery to order confirmation.
  • Build customer trust through transparent policies, clear product information, and excellent customer service.
  • Demonstrate modern web development best practices including semantic HTML, responsive CSS, and accessible design (WCAG 2.1).
  • Offer a scalable page structure that could be connected to a real back-end or CMS in a production environment.
  • Ensure fast load times and mobile-friendly layouts across all devices.

Site Structure

  • Main Pages: Home, About, Contact, Sitemap, Concept, Wireframes, Report
  • Shopping: Products listing, individual product detail pages (12 products across 6 pet categories)
  • Cart & Checkout: Shopping cart, checkout form, payment page, order confirmation, order status / tracking
  • Customer Service: Live chat support page, FAQ / Help Center, order tracking
  • Account: Customer account dashboard
  • Legal & Policies: Privacy Policy, Terms & Conditions

Technology Stack

2. Wireframe Mockups

Wireframe mockups live at wireframes.html and are accessible from the main navigation bar (Wireframes link) and from the footer Quick Links on every page of the site.

The page contains CSS-based wireframe diagrams for all major page layouts, built entirely with HTML and CSS — no image files or external tools required. Each wireframe uses a distinct set of custom CSS classes to simulate a structural layout diagram with labeled content regions.

Wireframes Provided

  • Homepage — hero banner, category grid, featured products, newsletter
  • Products Listing — filter sidebar, product grid
  • Product Detail — image, info panel, add-to-cart, reviews
  • Shopping Cart — line items, order summary
  • Checkout — shipping form, shipping method, order summary
  • Payment — payment method selection, billing details
  • Contact — contact form, company info
  • About — company story, team grid, values, locations
  • Customer Service — live chat interface
  • FAQ / Help Center — accordion sections
  • Order Status — tracking timeline
  • Account Dashboard — profile summary, recent orders

Wireframe CSS Classes Used

Each wireframe diagram is rendered using custom CSS classes scoped to the wireframes page:

.wf-page      — outer page container
.wf-box       — generic content block
.wf-hero      — hero/banner region
.wf-nav       — navigation bar region
.wf-footer    — footer region
.wf-sidebar   — sidebar column
.wf-grid      — content grid area
.wf-label     — text label inside a region

Typography

The entire site uses a single font stack defined in the body selector: font-family: Arial, Helvetica, sans-serif. No external web fonts are loaded, keeping page load times minimal.

Images

Product images are sourced from Unsplash (royalty-free). Category icons use Unicode emoji characters (no image files required). All <img> elements include descriptive alt text for accessibility. Product images are lazy-loaded using the loading="lazy" attribute to improve initial page performance.

5. CSS for All Formatting (No Inline Styles)

All visual formatting across the entire Pet Supply Plus site is handled by a single external stylesheet: css/styles.css. Every HTML page links to this stylesheet in its <head> section:

<link rel="stylesheet" href="css/styles.css">

No style="" inline attributes are used on any HTML element. The only exception is wireframes.html, which contains a page-scoped <style> block in the <head> to define the wireframe diagram helper classes — these are still not inline styles on HTML elements.

CSS Custom Properties (Design Tokens)

All design values (colors, spacing, typography, transitions) are defined as CSS custom properties in the :root selector at the top of css/styles.css:

:root {
  --color-primary:    #1B3A5C;   /* dark navy blue */
  --color-secondary:  #2AABB3;   /* teal */
  --color-accent:     #F5A623;   /* orange/gold */
  --color-white:      #FFFFFF;
  --color-bg:         #F5F5F5;
  --color-text:       #333333;
  --color-text-light: #666666;
  --container-max:    1200px;
  --border-radius:    4px;
  --transition:       0.3s ease;
}

Responsive Design

The CSS uses a mobile-first approach with three breakpoints. Navigation collapses to a hamburger menu on mobile, product grids stack to single columns, and the footer grid switches from 4 columns to a single column on small screens. All layout changes are handled entirely through CSS media queries — no JavaScript-driven style manipulation is used for layout.
